🏗️ Main System Parts

The system is built to be fast and safe. It focuses on catching dangerous files before they are saved to the library.

  • File Scanner — This is the system's "Bodyguard." Every file uploaded is checked in three ways to make sure it doesn't contain a virus or a harmful script. It can even peek inside a PDF to find hidden threats.
  • PDF Validator — This is like a "Reading Assistant." It reads through the uploaded manuscript to find specific pages, like the Approval Sheet. If it can't find them, it doesn't block the student—it just leaves a note for the Admin to double-check.
  • Attachment System — Instead of just one file, students can upload a whole set of extra files (like source code or data) that stays linked to their research project.

👨‍🎓 Student Experience

We designed the student process to be as smooth as possible. Even if the system can't find your signatures, it still lets you submit so a human can check them.

  • Easy Uploads: You won't be blocked just because the system can't "read" your signature sheet. If it looks like a scanned image, the system just tells the Admin to check it manually.
  • Safety First: The only thing that will stop an upload is a virus. This keeps everyone's computers safe.
  • Resubmitting: If an Admin asks for changes, you can jump back in, update your details, and add more files without starting over.

⚖️ Administrator Role

The Admin is the final human eye on the project. The system helps you by doing the "boring" work first.

  • Verification Report: Instead of searching for errors yourself, the system shows you a report. It tells you if the file is safe and if it found the required sections.
  • Final Approval: You have the final say. If the system gives a "Warning" because it can't read a signature, you can check it yourself and approve it anyway.
  • Archive Control: You can restore deleted projects or set deadlines for when submissions are allowed.
